# このリポジトリでClaudeに守ってもらうこと

このファイルは、Claude Code や Claude Desktop がこのリポジトリで作業するときに
自動で読み込まれます。運用スタッフが安全にAIを使うためのルールです。

## このリポジトリの構成

```
keywords.txt              記事にしたいキーワード（運用スタッフが編集する唯一のファイル）
content/*.md              生成された記事。AIが書き、人が確認する
site/                     公開用のHTML。自動生成なので手で編集しない
scripts/generate.py       記事生成。Claude APIを呼ぶ
scripts/build.py          content/ から site/ を作る
.github/workflows/        毎朝の自動実行の設定
```

## やってよいこと

- `keywords.txt` にキーワードを追記する
- `content/` の記事の文章を直す（誤字修正、表現の調整）
- `python scripts/build.py` を実行してサイトを作り直す
- `python scripts/generate.py` を実行して記事を1本追加する

## やってはいけないこと

- **`site/` の中を直接編集しない**。ビルドで上書きされるので、直しても消えます。
  記事を直したいときは `content/` の方を編集してから、ビルドし直してください。
- **APIキーをファイルに書かない**。`.env` にもコミットしない。
  キーは GitHub の Secrets か、手元の環境変数にだけ置きます。
- **記事を確認せずに公開しない**。AIは事実を間違えることがあります。
  特に統計の数字、企業名、制度名は必ず人間が裏を取ってください。
- **`.github/workflows/` を勝手に書き換えない**。自動実行が止まります。
  変更が必要なときは開発者に相談してください。

## 記事の品質について

生成された記事は、公開前に必ず次を確認してください。

1. 数字や固有名詞が実在するか（AIはもっともらしい嘘を書くことがあります）
2. 自社のサービス内容と矛盾していないか
3. 他社を不当に貶める表現がないか
4. 同じ内容の記事が既にないか

問題があれば `content/` の該当ファイルを直すか、削除してビルドし直してください。

## 記事のトーンを変えたいとき

`scripts/generate.py` の `SYSTEM_PROMPT` を書き換えてください。
ここが記事の書き方の指示です。変更後は1本生成して、意図通りか確認してから
本番の自動実行に任せてください。

## 困ったとき

- 記事が生成されない → `keywords.txt` に未処理のキーワード（# がついていない行）があるか確認
- サイトに反映されない → `python scripts/build.py` を実行したか確認
- 自動実行が動かない → GitHubの Actions タブでエラーを確認。API残高切れが多いです
