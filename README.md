# AIが毎朝記事を書くサイト

キーワードを1行書いておくと、毎朝7時にAIが記事を1本書き、
サイトが自動で更新されます。サーバー代・維持費は0円です。

## 何が0円なのか

| | 使うもの | 費用 |
|---|---|---|
| 記事を書く | Claude API | 1記事あたり約5〜15円 |
| 自動実行 | GitHub Actions | 無料枠内（1日1回なら余裕） |
| サイトの置き場 | 手持ちのサーバー | 既存のものを使う |

つまり**毎月かかるのは記事のAPI代だけ**です。毎日1本書いても月200〜400円程度。
サーバーを新しく借りる必要も、月額のサービスに入る必要もありません。

理由は、サイトが**ただのHTMLファイル**だからです。
サーバー側でプログラムが動いていないので、動かし続けるための費用が発生しません。
記事を作るのはGitHubの中だけで、出来上がったHTMLを置くだけになっています。

## 使い方

### 記事を増やしたいとき

`keywords.txt` に書きたいことを1行足すだけです。

```
地方 実証実験 誘致 自治体 課題    ← これが翌朝、記事になります
```

上から順に、1日1本ずつ記事になります。書き終わったキーワードには自動で
`#` がついて、次から飛ばされます。**普段触るのはこのファイルだけです。**

### いますぐ1本書かせたいとき

GitHubの「Actions」タブ →「記事の自動生成」→「Run workflow」を押してください。
数分で記事が増えます。

### 記事を直したいとき

`content/` の中の `.md` ファイルを直してから、ビルドし直します。

```
python scripts/build.py
```

`site/` の中を直接いじっても、次のビルドで消えるので注意してください。

### 公開するとき

`site/` の中身をまるごとエックスサーバーにアップロードするだけです。
Actionsの実行結果からzipでダウンロードすることもできます。

## 最初の設定（1回だけ）

1. このリポジトリをGitHubに置く
2. Settings → Secrets and variables → Actions → New repository secret
   - Name: `ANTHROPIC_API_KEY`
   - Value: Claude APIのキー
3. Settings → Actions → General → Workflow permissions
   - 「Read and write permissions」を選んで保存

これで翌朝から自動で動きます。

## 手元で試すとき

```
export ANTHROPIC_API_KEY=sk-ant-...
python scripts/generate.py    # 記事を1本書く
python scripts/build.py       # サイトを作る
```

`site/index.html` をブラウザで開けば確認できます。

## 記事のトーンを変えたいとき

`scripts/generate.py` の中の `SYSTEM_PROMPT` を書き換えてください。
ここが「どういう記事を書くか」の指示です。

## 注意

**AIが書いた記事は、公開前に人が確認してください。**
特に数字・企業名・制度名は、もっともらしい嘘が混ざることがあります。
詳しくは `CLAUDE.md` に書いてあります。
